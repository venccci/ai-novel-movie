## 1. 项目背景与立项动机

### 1.1 背景问题

剧本本质上是**“事件与对白的描述”**，但视觉生成系统需要的是：

- 明确的镜头边界
- 明确的画面构图
- 明确的镜头时长
- 明确的角色出场与动作

如果直接用剧本生成分镜图，常见问题包括：

- 一个画面塞太多信息
- 镜头节奏混乱
- 对白无处安放
- 后期剪辑无法控制时长

------

### 1.2 项目定位

本项目的定位是：

> **生成一份“机器 + 人类导演都能理解”的镜头清单（Shot List），
>  它是后续所有视觉生成的唯一权威输入。**

可以把它理解为：
 **AI 版导演分镜脚本（Director Script）**

------

## 2. 项目目标（Goals）

### 2.1 核心目标

1. 将 Script 中的 Scene 拆解为镜头级 Shot
2. 为每个镜头生成完整的拍摄参数
3. 保证整体节奏、时长与 StyleProfile 对齐
4. 为分镜图 / 视频生成提供“确定性输入”

------

### 2.2 成功指标（KPI）

| 指标         | 目标                     |
| ------------ | ------------------------ |
| 镜头可执行率 | ≥ 95% 镜头可直接生成画面 |
| 时长命中率   | 单集时长偏差 ≤ ±10%      |
| 对白覆盖率   | 100% 对白绑定镜头        |
| 人工调整率   | < 30% 镜头需要人工调整   |

------

## 6. 详细功能需求

------

## 6.1 Scene → Shot 拆分模块

### 6.1.1 拆分原则（非常关键）

一个 Shot 应满足：

- 单一视觉焦点
- 单一主要动作或情绪
- 可在 2–6 秒内表达清楚（可配置）

------

### 6.1.2 拆分策略

- 动作变化 → 新镜头
- 情绪变化 → 新镜头
- 视角变化 → 新镜头
- 对白切换 → 可新镜头（视节奏）

------

### 输出示例

```
{
  "shotId": "shot_01",
  "sceneId": "scene_03",
  "order": 1
}
```

------

## 6.2 镜头属性生成模块

### 6.2.1 摄影属性（Camera）

| 属性       | 示例                      |
| ---------- | ------------------------- |
| shotSize   | 特写 / 近景 / 中景 / 全景 |
| cameraMove | 静止 / 推 / 拉 / 摇 / 移  |
| angle      | 平视 / 仰视 / 俯视        |

------

### 6.2.2 构图属性（Composition）

- 角色站位（左 / 中 / 右）
- 视线方向
- 前景 / 背景层次
- 构图比例（三分法 / 居中）

------

### 6.2.3 情绪与动作

- 角色表情
- 氛围关键词（紧张 / 温馨 / 压抑）
- 关键动作描述（简洁）

------

### 6.2.4 时长生成

- 基于：
  - 镜头类型
  - 对白长度
  - StyleProfile 节奏
- 单镜头时长：1.5s – 8s（可配置）

------

## 6.3 台词与旁白绑定

### 6.3.1 绑定规则

- 每句对白 **必须** 绑定一个 Shot
- 可选展示方式：
  - 屏幕字幕
  - 旁白
  - 角色内心独白

------

### 示例

```
{
  "dialogueRef": "dlg_12",
  "display": "subtitle",
  "timing": "full"
}
```

------

## 6.4 ShotList 生成与管理

### 6.4.1 ShotList 结构定义（核心）

```
{
  "shotId": "shot_08",
  "sceneId": "scene_03",
  "order": 8,
  "characters": ["char_01", "char_02"],
  "camera": {
    "shotSize": "medium",
    "move": "static",
    "angle": "eye_level"
  },
  "composition": {
    "layout": "two_shot",
    "focus": "char_01"
  },
  "action": "主角回头",
  "emotion": "紧张",
  "dialogueRef": ["dlg_12"],
  "durationSec": 3.5,
  "imagePrompt": "A young man in school uniform turns back with a tense expression, medium shot, eye level angle, detailed anime style, vibrant colors",
  "flags": ["key_shot"]
}
```

------

### 6.4.2 镜头级表格输出格式（核心交付物）

最终输出的导演脚本将以表格形式呈现，每个镜头作为一个独立的行条目，便于导演和制作团队查阅与执行：

| 镜头序号 | 镜头ID | 场景ID | 角色 | 镜头规格 | 构图 | 动作描述 | 情绪 | 对白引用 | 时长(秒) | 图片生成提示词 | 标记 |
|----------|--------|--------|------|----------|------|----------|------|----------|----------|----------------|------|
| 1 | shot_01 | scene_01 | ["char_01"] | 特写/静止/平视 | 单人/聚焦主角 | 主角惊讶 | 惊讶 | dlg_01 | 2.5 | Close-up of a surprised young man in school uniform, anime style, expressive eyes, detailed facial features | key_shot |
| 2 | shot_02 | scene_01 | ["char_01", "char_02"] | 中景/静止/平视 | 双人/主角在左 | 主角与配角对峙 | 紧张 | dlg_02-dlg_05 | 4.2 | Medium shot of two characters facing each other in school uniforms, anime style, tense atmosphere, dynamic poses | - |
| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |

**表格说明**：
- 每一行代表一个独立的镜头，包含该镜头的所有关键属性
- **图片生成提示词**列提供用于AI图像生成的具体提示词，确保视觉一致性
- **每个镜头都必须包含对应的图片生成提示词**，用于驱动AI图像生成模型，确保画面与导演意图一致
- 图片生成提示词基于镜头属性（镜头规格、构图、动作、情绪等）自动生成，并融合Style Profile中的视觉风格配置
- 所有字段均来自ShotList JSON结构，以表格形式提供更直观的视角
- 表格支持导出为CSV/Excel格式，便于团队协作与进度跟踪

| -------- | -------- |

| locked   | 下游使用 |

## 6.5 导演备注（Director Notes）

- 可选自由文本

## 7. 关键校验规则（工业级）

### 7.1 自动校验

- Σ durationSec ≈ 单集时长
- 所有对白均被绑定
- 相邻镜头角色连续性合理
- 镜头密度符合 StyleProfile

------

### 7.2 阻断规则

- ShotList 未锁定 → 禁止进入分镜图生成
- 对白未绑定 → 红色阻断
- 时长严重超标 → 强制重生成

------

## 8. 人工介入设计（不是补锅，是创作控制）

- 合并 / 拆分镜头
- 手动改镜头时长
- 标记关键镜头（封面 / 高潮）
- 锁定镜头防止 AI 覆盖

------

## 9. 非功能需求（NFR）

| 维度   | 要求             |
| ------ | ---------------- |
| 稳定性 | 同输入生成一致   |
| 可解释 | 每镜头可回溯来源 |
| 扩展性 | 支持新镜头类型   |
| 性能   | 单集生成 < X 秒  |
