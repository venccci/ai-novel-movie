# AI漫剧生成平台 - 交互说明

## 概述

AI漫剧生成平台是一个端到端的AI创作工具，将小说文本转化为可发布的漫剧视频。平台采用严格的四步工作流程，确保每个步骤的输出质量并为后续步骤提供稳定的输入。

**核心设计原则：**
- **严格顺序执行**：四个步骤必须按顺序完成，前一步骤完成后才能开始下一步
- **状态锁定**：每个步骤完成后，其输出产物处于“锁定”状态，不可被修改
- **流程可视化**：提供清晰的流程状态看板，展示每个步骤的完成状态与阻塞点
- **用户可控**：在关键节点提供人工介入点，增强而非修补AI输出

## 用户旅程

### 1. 项目创建与风格设定（第一步）
**用户目标**：创建新项目并定义全局视觉风格

**用户操作流程**：
1. 点击“创建新项目”按钮
2. 填写项目基本信息：
   - 项目名称（必填）
   - 项目描述（可选）
   - 目标平台（抖音/快手/B站/YouTube）
   - 项目语言（中文/多语言）
3. 选择风格模板：
   - 预设模板（国漫·热血、日漫·校园、写实·剧情、Q版·轻喜剧）
   - 自定义风格
4. 配置视觉规格：
   - 画面比例 (9:16 / 16:9)
   - 分辨率 (1080×1920)
   - 帧率 (24/30)
   - 艺术风格 (国漫/日漫/写实)
   - 渲染风格 (线稿/厚涂)
5. 配置叙事与节奏参数：
   - 单集时长（秒）
   - 镜头密度（镜头/分钟）
   - 旁白占比
   - 对白占比
   - 节奏（快/中/慢）
6. 预览并确认配置
7. 锁定Style Profile，进入下一步

**交互状态**：
- `草稿态`：可自由修改所有配置
- `已保存`：配置已保存但可修改
- `已锁定`：配置锁定，不可修改，下游步骤可读取

### 2. 小说剧本生成（第二步）
**前提条件**：第一步Style Profile必须处于“已锁定”状态

**用户目标**：将小说文本转化为结构化剧本

**用户操作流程**：
1. 上传小说文本：
   - 文本粘贴
   - 文件上传 (txt/docx/md)
   - URL导入（可选）
2. 文本清洗与标准化：
   - 自动去除广告段落、作者注释
   - 标点统一（全角/半角）
   - 人名与称谓归一化
3. 章节/事件切分：
   - 自动识别章节
   - 手动调整切分点
   - 跨章节合并为一集
4. 剧本结构化生成：
   - Scene（场次）自动识别
   - 对白/旁白抽取
   - 角色情绪标注
5. 多版本生成与管理：
   - 生成不同节奏版本（快/中/慢）
   - 生成不同风格版本（情绪强化/悬疑强化）
   - 预览并选择主版本
6. 人工介入与调整：
   - Scene合并/拆分
   - 对白改写
   - 情绪调整
7. 锁定剧本，进入下一步

**交互状态**：
- `处理中`：AI正在处理小说文本
- `已生成`：剧本已生成，可预览
- `已调整`：人工调整已完成
- `已锁定`：剧本锁定，供下游步骤使用

### 3. 角色设计与资产库（第三步）
**前提条件**：第二步Script必须处于"已锁定"状态

**用户目标**：从剧本中生成角色资产，确保角色一致性

**用户操作流程**：
1. 角色自动抽取：
   - 从剧本中识别所有角色
   - 自动分类（主角/配角/路人）
   - 显示角色出现频次统计
2. 角色设定卡生成：
   - 自动生成角色基础信息（姓名、年龄、性别）
   - 自动生成角色外形描述（身高、体型、发型）
   - 自动生成服装与特征物
3. 角色视觉生成：
   - 基于Style Profile生成角色视觉
   - 生成多个候选版本（正面、侧面、半身、表情集）
   - 自动相似度聚类，淘汰离群图
4. 人工选择与调整：
   - 从候选版本中选择最符合的角色
   - 手动调整角色特征
   - 锁定角色一致性锚点（发型、服装、颜色、面部特征）
5. 角色资产入库：
   - 角色设定卡存储
   - 参考图集合管理
   - 一致性锚点定义
6. 锁定角色资产，进入下一步

**交互状态**：
- `抽取中`：正在从剧本抽取角色
- `设定生成中`：正在生成角色设定卡
- `视觉生成中`：正在生成角色视觉
- `待选择`：候选版本已生成，等待用户选择
- `已锁定`：角色资产锁定，供下游步骤使用

### 4. 镜头级导演脚本生成（第四步）
**前提条件**：第二步Script和第三步Character Assets必须处于"已锁定"状态

**用户目标**：将剧本转化为可执行的镜头级导演脚本

**用户操作流程**：
1. Scene → Shot自动拆分：
   - 基于Scene结构自动拆分镜头
   - 遵循单一视觉焦点原则
   - 镜头时长范围：1.5s - 8s（可配置）
2. 镜头属性生成：
   - 摄影属性（镜头大小、运镜方式、角度）
   - 构图属性（角色站位、视线方向、前景背景）
   - 情绪与动作描述
   - 对白绑定
3. 图片生成提示词生成：
   - 基于镜头属性自动生成图片提示词
   - 融合Style Profile视觉风格
   - 确保角色一致性约束
4. 导演脚本表格生成：
   - 每个镜头作为独立行条目
   - 包含所有镜头属性
   - 包含图片生成提示词
5. 人工介入与调整：
   - 镜头合并/拆分
   - 镜头时长调整
   - 标记关键镜头（封面/高潮）
6. 锁定导演脚本，输出最终结果

**交互状态**：
- `拆分中`：正在拆分Scene为Shot
- `属性生成中`：正在生成镜头属性
- `提示词生成中`：正在生成图片提示词
- `已生成`：导演脚本已生成，可预览
- `已调整`：人工调整已完成
- `已锁定`：导演脚本锁定，供下游系统使用

## 状态管理与转换

### 全局状态定义
```javascript
const projectStates = {
  DRAFT: 'draft',      // 项目草稿态
  IN_PROGRESS: 'in_progress',  // 项目进行中
  LOCKED: 'locked',    // 项目已完成并锁定
  ARCHIVED: 'archived' // 项目已归档
};

const stepStates = {
  LOCKED: 'locked',    // 步骤未解锁（前置条件未满足）
  READY: 'ready',      // 步骤可开始（前置条件已满足）
  PROCESSING: 'processing',  // 步骤处理中
  REVIEW: 'review',    // 步骤待审核
  COMPLETED: 'completed',    // 步骤已完成
  ERROR: 'error'       // 步骤执行失败
};
```

### 状态转换规则
1. **项目创建**：空项目 → `DRAFT`
2. **步骤启动**：当前步骤为`READY`且用户点击开始 → `PROCESSING`
3. **AI处理完成**：`PROCESSING` → `REVIEW`
4. **用户确认**：`REVIEW` → `COMPLETED`
5. **步骤锁定**：`COMPLETED` → 触发下一步骤状态变为`READY`
6. **处理失败**：`PROCESSING` → `ERROR`
7. **重试处理**：`ERROR` → `PROCESSING`

### 前置条件检查机制
系统在允许用户启动任一步骤前，必须检查：
1. 所有前置步骤的状态是否为`COMPLETED`
2. 前置步骤的输出产物是否处于`LOCKED`状态
3. 当前用户是否有权限操作该步骤

## 错误处理与用户反馈

### 错误类型分类
1. **输入验证错误**：用户输入不符合要求
   - 反馈方式：表单字段内错误提示，红色边框+错误信息
   - 处理方式：阻止提交，高亮错误字段

2. **AI处理错误**：AI模型处理失败
   - 反馈方式：Toast通知+错误详情链接
   - 处理方式：自动重试机制，提供重试按钮

3. **系统错误**：后端服务或网络问题
   - 反馈方式：全屏错误页面，提供技术支持联系方式
   - 处理方式：保存用户操作，支持恢复

4. **数据一致性错误**：步骤间数据不一致
   - 反馈方式：流程状态看板警告图标+详细说明
   - 处理方式：提供数据修复工具

### 用户反馈机制
1. **即时反馈**：操作后立即给出视觉反馈（按钮状态、加载动画）
2. **进度反馈**：长时间操作显示进度条或加载状态
3. **结果反馈**：操作完成后显示成功/失败通知
4. **确认反馈**：重要操作前要求用户确认

### 表单验证最佳实践
1. **即时验证策略**：
   - 字段失焦时验证（onBlur）：大多数字段在用户离开字段时验证
   - 实时验证（onChange）：关键字段（如密码强度）可实时验证
   - 避免仅提交时验证：用户提交前应知道所有问题

2. **错误消息设计**：
   - **位置**：错误消息显示在问题字段附近
   - **清晰度**：明确说明问题及解决方法
   - **可访问性**：使用`role="alert"`或`aria-live`让屏幕阅读器播报错误
   - **视觉指示**：红色边框+图标+文字说明的组合

3. **提交反馈**：
   - **加载状态**：提交按钮显示加载状态，禁用重复提交
   - **成功反馈**：清晰的成功消息，指示下一步操作
   - **错误恢复**：提供明确的恢复路径（重试按钮、帮助链接）

4. **表单标签与占位符**：
   - **必用标签**：所有表单控件必须有关联的`<label>`
   - **避免占位符替代**：占位符不能替代标签，仅作为辅助提示
   - **标签位置**：标签位于输入框上方或左侧，易于关联

5. **错误恢复支持**：
   - **恢复路径**：每个错误都应提供明确的恢复步骤
