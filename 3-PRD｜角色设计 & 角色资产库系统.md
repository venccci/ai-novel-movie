## 1. 项目背景与立项动机

### 1.1 背景问题

在 AI 视觉生成中，**角色一致性是失败率最高的环节**：

- 同一角色多镜头“换脸 / 换发型 / 换衣服”
- 人物设定复杂 → 模型不稳定
- 没有角色资产 → 每一帧都是“重新发明角色”

如果没有一个**独立的角色资产系统**，后续的分镜、视频、剪辑都会陷入不可控状态。

------

### 1.2 项目定位

本项目不是“画几个人”，而是：

> **把小说中的“角色概念”，升级为 AI 系统可稳定复用的“角色资产”。**

它的本质是一个 **Character-as-Asset（角色即资产）系统**。

------

## 2. 项目目标（Goals）

### 2.1 核心目标

1. 从剧本中自动抽取角色并分类（主 / 配 / 路人）
2. 为每个角色生成 **标准化角色设定卡**
3. 为每个角色生成 **可复用、可约束的一致性资产**
4. 为下游模块提供 **唯一角色引用源（Character ID）**

------

### 2.2 成功指标（KPI）

| 指标       | 目标                          |
| ---------- | ----------------------------- |
| 角色一致性 | 同一角色跨 10 个镜头外观一致  |
| 生成稳定性 | 单角色生成失败率 < 10%        |
| 复用率     | 同一角色资产被 ≥ N 个镜头引用 |
| 人工介入率 | < 30% 角色需要人工锁定        |

------

## 3. 用户画像与使用场景

### 3.1 用户画像

| 用户        | 使用目的                 |
| ----------- | ------------------------ |
| 内容创作者  | 快速得到“稳定能用”的角色 |
| 编剧 / 策划 | 确认角色形象是否符合设定 |
| AI 运营     | 降低重生成成本           |
| 下游系统    | 稳定引用角色             |

------

### 3.2 核心使用场景

- 从剧本一键生成角色资产
- 人工挑选“最像”的角色版本
- 锁定角色关键特征供后续镜头复用
- 在多个分镜、视频中反复调用同一角色

## 6. 详细功能需求

------

## 6.1 角色清单抽取模块

### 6.1.1 角色抽取规则

从 Script 中自动提取：

- 出现频次
- 是否有对白
- 是否参与关键 Scene

### 6.1.2 角色分类

| 类型 | 判定规则            |
| ---- | ------------------- |
| 主角 | 贯穿剧情 / 高频出场 |
| 配角 | 多次出现但非主线    |
| 路人 | 只在个别场景出现    |

------

### 输出

```
{
  "characterId": "char_01",
  "roleType": "main",
  "mentions": 42
}
```

------

## 6.2 角色设定卡生成（Character Sheet）

### 6.2.1 角色设定字段

| 类别 | 字段                 |
| ---- | -------------------- |
| 基础 | 姓名 / 年龄 / 性别   |
| 外形 | 身高 / 体型 / 发型   |
| 服装 | 主服装 / 配饰        |
| 性格 | 性格关键词           |
| 标志 | 特征物 / 伤疤 / 习惯 |

------

### 6.2.2 CharacterSheet 示例

```
{
  "characterId": "char_01",
  "name": "林凡",
  "age": 18,
  "bodyType": "slim",
  "hair": "black short hair",
  "outfit": "school uniform",
  "personality": ["calm", "determined"],
  "signature": "left wrist bracelet"
}
```

------

## 6.3 角色视觉生成模块

### 6.3.1 视觉生成策略

- 基于 CharacterSheet + StyleProfile
- 生成多个候选版本（N 张）
- 覆盖：
  - 正面
  - 侧面
  - 半身
  - 表情集（喜怒哀乐）

------

### 6.3.2 候选筛选

- 相似度聚类
- 自动淘汰离群图
- 人工选择最终版本（可选）

------

## 6.4 角色一致性锚点（核心）

### 6.4.1 锚点定义

**锚点 = 后续生成中“不可被模型随意改动”的特征**

| 类型 | 示例        |
| ---- | ----------- |
| 发型 | 黑色短发    |
| 服装 | 校服+白衬衫 |
| 颜色 | 主色调      |
| 面部 | 脸型、瞳色  |

------

### 6.4.2 锚点形式

- Prompt 固定片段
- 参考图集合
- 模型权重（LoRA / embedding，预留）

------

## 6.5 角色资产入库（Character Assets）

### 6.5.1 资产内容

- Character ID（全局唯一）
- 角色版本号
- 角色设定卡
- 参考图（多张）
- 一致性锚点定义

------

### 6.5.2 资产版本规则

- 任意修改 → 新版本
- 已锁定版本不可修改
- 下游只能引用锁定版本

------

## 6.6 人名别名映射（Character Dictionary）

### 6.6.1 映射规则

```
{
  "char_01": ["林凡", "阿凡", "他", "少年"]
}
```

- 用于 Script / 分镜 / 字幕阶段统一角色引用
- 保证角色 ID 不漂移

------

## 7. 核心输出定义

### 7.1 CharacterAssets 结构

```
{
  "characterId": "char_01",
  "version": "v1.0",
  "characterSheet": {...},
  "anchors": {...},
  "referenceImages": [],
  "status": "locked"
}
```

------

## 8. 关键校验规则

### 8.1 自动校验

- 多图相似度阈值 ≥ X
- 角色特征是否缺失
- 设定复杂度是否超标

------

### 8.2 阻断规则

- 未锁定角色 → 不允许进入分镜生成
- 角色设定缺失关键字段 → 标红

------

## 9. 人工介入设计（必须“可控”）

- 选择候选版本
- 锁定/解锁锚点
- 删除异常角色
- 合并角色（如前期拆错）

------

## 10. 非功能需求（NFR）
