# RFC-002-小说剧本生成系统

## 标题

小说剧本生成系统

## 摘要

实现小说文本到可拍可演的结构化剧本，包括小说导入、文本清洗、章节/事件切分、剧情结构化、对白/旁白抽取和剧本版本管理。

## 验收标准

### 小说导入模块

- 用户可以导入小说文本，支持文本粘贴、文件上传（txt / docx / md）和URL导入
- 系统可以自动识别章节，用户可以手动选择生成范围
- 系统可以保存原始文本和章节信息

### 文本清洗与标准化

- 系统可以自动清洗文本，去广告段落、去作者注释、去空行、重复符号和标点统一
- 系统可以自动抽取角色候选，建立CharacterAliasMap，后续解析统一使用character_id

### 章节 / 事件切分

- 系统可以自动切分章节和事件，优先按章节，章节过长时按事件点切分
- 单集目标长度受StyleProfile约束

### 剧情结构化

- 系统可以自动生成Scene，包含地点、时间、出场人物、核心事件和情绪基调

### 对白与旁白提取

- 系统可以自动解析对白，包含说话人、对象、情绪和是否为内心独白
- 系统可以自动解析旁白，包含描述环境、补充心理活动和后续可用于TTS

### 剧本版本管理

- 用户可以选择不同的版本策略，系统生成不同版本的剧本（快节奏版、情绪强化版、悬疑强化版）
- 系统可以管理剧本的版本状态，包括草稿、人工调整、主线版本和废弃

### 剧本输出定义

- 系统可以根据用户输入的参数，生成结构化的Script.json
- 系统可以自动生成ScriptSummary，包含整体剧情摘要、角色关系说明和本集看点

### 关键校验规则

- 系统可以自动校验剧本，包括Scene是否完整、对白是否超长、情绪字段是否缺失和同一角色称谓是否统一
- 系统可以根据校验结果，阻断不符合要求的剧本进入下一项目，无Scene不可进入下一项目，对白未绑定角色标红，超出目标时长提示重生成

### 人工介入点

- 用户可以对一句话进行改写，系统保存改写后的文本
- 用户可以删除或合并Scene，系统保存修改后的剧本
- 用户可以锁定某句对白不再被AI改写，系统保存锁定状态
- 用户可以选择一个版本作为主版本，系统保存主版本信息

## API接口

### 导入小说

```http
POST /api/v1/projects/{id}/novels
Content-Type: multipart/form-data

{
  "file": "小说文件",
  "styleProfileId": "style-profile-id"
}
```

### 获取小说列表

```http
GET /api/v1/projects/{id}/novels
```

### 获取小说详情

```http
GET /api/v1/projects/{id}/novels/{id}
```

### 生成剧本

```http
POST /api/v1/projects/{id}/novels/{id}/scripts
Content-Type: application/json

{
  "versionStrategy": "快节奏版"
}
```

### 获取剧本列表

```http
GET /api/v1/projects/{id}/novels/{id}/scripts
```

### 获取剧本详情

```http
GET /api/v1/projects/{id}/novels/{id}/scripts/{version}
```

### 更新剧本

```http
PUT /api/v1/projects/{id}/novels/{id}/scripts/{version}
Content-Type: application/json

{
  "scenes": [
    {
      "sceneId": "scene-03",
      "location": "街道",
      "time": "夜晚",
      "characters": ["char_01", "char_02"],
      "conflict": "追逐",
      "turningPoint": "主角被拦下",
      "emotion": "紧张"
    }
  ]
}
```

### 锁定剧本

```http
POST /api/v1/projects/{id}/novels/{id}/scripts/{version}/lock
```

### 解锁剧本

```http
POST /api/v1/projects/{id}/novels/{id}/scripts/{version}/unlock
```

## 安全要求

### 身份验证

- 使用 JWT 进行身份验证
- 令牌过期时间设置为 1 小时
- 刷新令牌过期时间设置为 7 天

### 授权

- 使用 RBAC（基于角色的访问控制）进行授权
- 定义三种角色：管理员、开发者、用户
- 不同角色拥有不同的权限

### 数据加密

- 使用 HTTPS 进行数据传输
- 数据库密码使用 bcrypt 进行加密
- 敏感数据使用 AES 进行加密
