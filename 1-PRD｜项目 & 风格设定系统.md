## 1. 项目背景与立项动机

### 1.1 背景问题

在 AI 内容生成系统中，如果**缺少统一的项目级与风格级配置**，将导致：

- 各阶段 AI 输出风格不一致
- 不同模型“各自发挥”，难以控制
- 重跑成本高，无法复现结果
- 无法批量生产、无法规模化

尤其在 **小说 → 漫剧视频** 场景中，涉及文本、图像、视频、音频多个模型，**必须存在一个“全局约束层”**。

------

### 1.2 项目定位

本项目是 **整个 AI 漫剧流水线的起点与中枢控制系统**，负责：

> 把“用户的模糊意图”
>  → 转换成“所有 AI 模型可执行的、统一的、结构化的约束配置”

------

## 2. 项目目标（Goals）

### 2.1 核心目标

1. 提供 **标准化的项目创建与管理能力**
2. 提供 **统一、可复用、可版本化的风格描述体系（Style Profile）**
3. 为后续所有 AI 项目提供 **唯一可信的全局配置来源（Single Source of Truth）**

------

### 2.2 成功指标（KPI）

| 指标           | 说明                                   |
| -------------- | -------------------------------------- |
| 风格一致性     | 同一项目内多阶段输出风格一致           |
| 复现能力       | 同一配置多次运行结果稳定               |
| 下游引用成功率 | 下游模块读取 StyleProfile 成功率 ≥ 99% |
| 项目创建时长   | 用户完成一次项目配置 ≤ 3 分钟          |

## 6. 详细功能需求

------

### 6.1 项目管理模块

#### 6.1.1 创建项目

**功能描述**
 用户创建一个新的漫剧生成项目。

**输入字段**

- 项目名称（必填）
- 项目描述（可选）
- 目标平台（抖音 / 快手 / B站 / YouTube）
- 项目语言（中文 / 多语言）

**输出**

- project_id
- 项目基础元信息

**业务规则**

- 项目名称唯一（同一用户下）
- 创建即生成默认 Style Profile（草稿态）

------

#### 6.1.2 项目状态

| 状态     | 说明             |
| -------- | ---------------- |
| Draft    | 草稿，允许修改   |
| Locked   | 锁定，供下游使用 |
| Archived | 归档             |

------

### 6.2 风格模板模块

#### 6.2.1 预设风格模板

**示例**

- 国漫 · 热血
- 日漫 · 校园
- 写实 · 剧情
- Q版 · 轻喜剧

**模板包含**

- 视觉风格预设
- 镜头密度建议
- 情绪强度范围

------

#### 6.2.2 模板应用规则

- 应用模板 = 初始化 Style Profile
- 模板应用后允许用户覆盖字段
- 模板本身不可被普通用户修改

------

### 6.3 风格配置模块（核心）

> 所有字段最终都会汇总进 Style Profile

#### 6.3.1 视觉规格配置

| 字段         | 示例               |
| ------------ | ------------------ |
| aspect_ratio | 9:16 / 16:9        |
| resolution   | 1080×1920          |
| frame_rate   | 24 / 30            |
| art_style    | 国漫 / 日漫 / 写实 |
| render_style | 线稿 / 厚涂        |

------

#### 6.3.2 叙事与节奏配置

| 字段             | 说明           |
| ---------------- | -------------- |
| episode_duration | 单集时长（秒） |
| shots_per_minute | 镜头密度       |
| narration_ratio  | 旁白占比       |
| dialogue_ratio   | 对白占比       |
| pacing           | 快 / 中 / 慢   |

------

#### 6.3.3 AI 行为约束（高级）

| 字段              | 说明           |
| ----------------- | -------------- |
| consistency_level | 角色一致性要求 |
| creativity_level  | 创意自由度     |
| motion_intensity  | 运镜强度       |
| retry_policy      | 失败重试策略   |

**默认值与推荐配置（实现约束）**

- 所有数值型字段必须提供系统推荐默认值，用于快速开箱即用：
  - `episode_duration`：默认 3600 秒
  - `shots_per_minute`：默认每 10 分钟 60 个镜头（内部以统一口径折算）
  - `narration_ratio`：默认 20%
  - `dialogue_ratio`：默认 40%
  - `pacing`：默认 `medium`
  - `consistency_level`：默认 80
  - `creativity_level`：默认 60
  - `motion_intensity`：默认 50
- 提示词类字段需要提供“推荐示例”，在 UI 中作为默认占位/初始值：
  - 正向提示：高质量、分辨率、镜头感等通用推荐组合
  - 负向提示：常见低质量/瑕疵项黑名单
- 项目名称为必填字段，不提供默认值，但允许用户先用占位名称创建后再修改

------

### 6.4 Style Profile 输出（核心产物）

#### 6.4.1 Style Profile 定义

**这是本项目的最终、最重要交付物**

```
{
  "projectId": "xxx",
  "version": "v1.0",
  "visual": {...},
  "narrative": {...},
  "aiConstraints": {...},
  "platform": {...}
}
```

------

#### 6.4.2 设计原则

- 完全结构化
- 无自然语言歧义
- 可直接作为 Prompt 输入
- 可持久化、可版本化

------

### 6.5 版本管理

- 每次修改生成新版本
- 支持回滚
- 下游模块必须绑定具体版本号

------

### 6.6 对外接口（供下游使用）

- `GET /projects/{id}/style-profile`
- `GET /projects/{id}/style-profile?version=v1.0`

**规则**

- Locked 状态才能被下游读取
- Draft 状态不可执行生成任务

## 11. 交互与视觉统一性要求（补充）

- 项目设定界面中所有配置选项（按钮组、标签、输入框、滑块、下拉选择等）必须复用统一的 UI 组件库：
  - 标签/说明文字统一使用 `InputGroup` 风格的标题 + 描述排版
  - 行为按钮统一使用全局 `Button` 组件的样式规范（主按钮/次按钮/强调按钮）
  - 弹窗统一使用 `Modal` 组件，保持遮罩、圆角、边框与动效一致
- 新增/扩展字段时，不允许在局部页面自定义与全局不一致的“特例样式”，以保证整体观感统一。

## 7. 非功能需求（NFR）

| 类别   | 要求               |
| ------ | ------------------ |
| 稳定性 | 99.9% 可用         |
| 一致性 | 同版本结果可复现   |
| 可扩展 | 新字段不破坏旧版本 |

## 8. 异常与边界场景

- 用户修改风格但已有任务在运行 → 新任务用新版本
- 下游请求不存在版本 → 返回错误并阻断
- 模板升级 → 不影响已有项目
