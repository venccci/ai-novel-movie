## 验收标准

### 项目管理模块

- 用户可以创建项目，输入项目名称、描述、目标平台和语言
- 项目名称唯一，同一用户下不能重复
- 创建项目后，系统生成默认的Style Profile（草稿态）
- 用户可以查看项目状态，包括草稿、锁定和归档
- 用户可以编辑项目信息，包括项目名称、描述、目标平台和语言
- 用户可以复制项目作为模板

### 风格模板模块

- 用户可以选择预设的风格模板，包括国漫·热血、日漫·校园、写实·剧情、Q版·轻喜剧
- 应用模板后，系统初始化Style Profile
- 用户可以修改模板中的字段，系统保存修改后的Style Profile
- 用户可以创建自定义的风格模板，输入风格模板的名称、描述和参数

### 风格配置模块

- 用户可以配置视觉规格，包括宽高比、分辨率、帧率、艺术风格、渲染风格
- 用户可以配置叙事与节奏参数，包括单集时长、镜头密度、旁白占比、对白占比、节奏
- 用户可以配置AI行为约束，包括角色一致性要求、创意自由度、运镜强度、失败重试策略

### Style Profile输出

- 系统可以根据用户输入的参数，生成结构化的Style Profile
- 系统可以保存Style Profile的多个版本，支持回滚和下游模块绑定具体版本号
- 下游模块可以通过API读取Style Profile，只有锁定状态的Style Profile可以被读取

## API接口

### 创建项目

```http
POST /api/v1/projects
Content-Type: application/json

{
  "name": "我的第一个漫剧项目",
  "description": "这是我的第一个漫剧项目",
  "targetPlatform": "抖音",
  "language": "中文"
}
```

### 获取项目列表

```http
GET /api/v1/projects
```

### 获取项目详情

```http
GET /api/v1/projects/{id}
```

### 更新项目

```http
PUT /api/v1/projects/{id}
Content-Type: application/json

{
  "name": "我的第一个漫剧项目（更新）",
  "description": "这是我的第一个漫剧项目（更新）",
  "targetPlatform": "B站",
  "language": "中文"
}
```

### 删除项目

```http
DELETE /api/v1/projects/{id}
```

### 创建Style Profile

```http
POST /api/v1/projects/{id}/style-profiles
Content-Type: application/json

{
  "version": "v1.0",

    "aspectRatio": "9:16",

    "frameRate": 24,

    "renderStyle": "厚涂"
```


### 更新Style Profile

```http

Content-Type: application/json

  "visual": {
```

```http

```

## 安全要求

### 身份验证

- 使用 JWT 进行身份验证
- 令牌过期时间设置为 1 小时
- 刷新令牌过期时间设置为 7 天

### 授权

- 使用 RBAC（基于角色的访问控制）进行授权
- 定义三种角色：管理员、开发者、用户
- 不同角色拥有不同的权限

### 数据加密

- 使用 HTTPS 进行数据传输
- 数据库密码使用 bcrypt 进行加密
- 敏感数据使用 AES 进行加密
