# RFC-003-角色设计&角色资产库系统

## 标题

角色设计&角色资产库系统

## 摘要

## 验收标准

### 角色清单抽取模块

- 系统可以自动从Script中提取角色，包含出现频次、是否有对白和是否参与关键Scene
- 系统可以自动对角色进行分类，分为主角、配角和路人

### 角色设定卡生成

- 系统可以自动生成标准化的角色设定卡，包含姓名、年龄、性别、身高、体型、发型、主服装、配饰、性格关键词和特征物

### 角色视觉生成模块

- 系统可以基于CharacterSheet + StyleProfile生成多个候选版本（正面、侧面、半身、表情集）
- 系统可以自动进行相似度聚类，自动淘汰离群图，用户可以选择最终版本

### 角色一致性锚点

- 系统可以自动定义锚点（发型、服装、颜色、面部），后续生成中不可被模型随意改动
- 锚点形式包括Prompt固定片段、参考图集合和模型权重（LoRA / embedding，预留）

### 角色资产入库

- 系统可以将角色资产（Character ID、角色版本号、角色设定卡、参考图、一致性锚点定义）入库
- 任意修改生成新版本，已锁定版本不可修改，下游只能引用锁定版本

### 人名别名映射

- 系统可以自动建立人名别名映射，用于Script / 分镜 / 字幕阶段统一角色引用

## API接口

### 生成角色

```http
POST /api/v1/projects/{id}/scripts/{version}/characters
Content-Type: application/json

{
  "styleProfileId": "style-profile-id"
}
```

### 获取角色列表

```http
GET /api/v1/projects/{id}/scripts/{version}/characters
```

### 获取角色详情

```http
GET /api/v1/projects/{id}/scripts/{version}/characters/{characterId}
```

### 更新角色

```http
PUT /api/v1/projects/{id}/scripts/{version}/characters/{characterId}
Content-Type: application/json

{
  "characterSheet": {
    "name": "林凡",
    "age": 18,
    "bodyType": "slim",
    "hair": "black short hair",
    "outfit": "school uniform",
    "personality": ["calm", "determined"],
    "signature": "left wrist bracelet"
  }
}
```

### 锁定角色

```http
POST /api/v1/projects/{id}/scripts/{version}/characters/{characterId}/lock
```

### 解锁角色

```http
POST /api/v1/projects/{id}/scripts/{version}/characters/{characterId}/unlock
```

### 身份验证

- 使用 JWT 进行身份验证
- 令牌过期时间设置为 1 小时
- 刷新令牌过期时间设置为 7 天

### 授权

- 使用 RBAC（基于角色的访问控制）进行授权
- 定义三种角色：管理员、开发者、用户
- 不同角色拥有不同的权限

### 数据加密

- 使用 HTTPS 进行数据传输
- 数据库密码使用 bcrypt 进行加密
- 敏感数据使用 AES 进行加密
